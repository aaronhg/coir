## Verify levels (honest labelling)

- **native-readback** — the engine reports the exact value coir wrote (strongest). Used for P2 (name/active/pos/euler) and RI1 (string), and for the **null→resolved** transitions in P1-1/P3b1.
- **native-survival** — the engine `reimport`s the edited file with no error (a malformed `cc.TargetOverrideInfo`/override would be rejected). The load-bearing native gate for set-ref correctness pairs this with the deterministic **offline** structure (the inline `{__id__}` / the override `localID`), because a resolved node-ref reads back only as `{uuid:<runtime id>}` — non-deterministic, so it proves *liveness*, not *which* node (see COVERAGE A).
- **cli-exit** — exit code + file bytes unchanged (refusals / dry-run). No editor.
- **cli-reread** — coir's own `-o json` / `get` shape (offline structural assertion).

### Findings

> **⚠ FINDING-A — the native node-ref readback proves liveness, not target identity (methodology, not a coir bug).**
> A resolved `cc.Node` reference reads back as `{uuid:<runtime id>}`, and that uuid is regenerated on every `cc.instantiate` (observed: two reads of the *same* untouched prefab gave different uuids). So a `set-ref` native readback can only assert **resolved vs `null`**, never *which* node. Target-correctness is therefore proven by the **offline** structure (the deterministic inline `{__id__:N}` / override `localID`, asserted via `-o json`/`get`), with the native layer proving import-survival + liveness. N1 and N3 are the strong exceptions: their field starts `null`/inline-`null`, so a non-null readback *is* falsifiable. Severity: n/a (a property of the engine's runtime, documented so future rounds don't over-claim).

> **⚠ FINDING-B — CLI `-o json` omitted `reimportReason` (minor parity gap) — FIXED.**
> A `needsReimport` edit's `-o json` carried `needsReimport:true` but **not** `reimportReason` (the reason string was printed only in text mode, while the MCP `commitResult` *does* include it). **Fixed** in `editCli.js applyResult` (the JSON now includes `reimportReason` when set), with a `test/cli.test.js` assertion. Severity: low.
